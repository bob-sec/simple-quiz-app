# クイズアプリ

リアルタイムクイズアプリケーション。管理者がタイミングを制御しながら進行するライブクイズシステムです。

## アーキテクチャ

| サービス | 技術 | 役割 |
|---------|------|------|
| `nginx` | Nginx | リバースプロキシ / TLS終端 |
| `frontend` | React + Vite + Nginx | クイズUI (スマートフォン対応) |
| `backend` | FastAPI + Python | REST API + WebSocket |
| `redis` | Redis | クイズ状態・回答データ保存 |

---

## クイックスタート

### 1. 環境変数の設定

```bash
cp .env.example .env
# .env を編集して ADMIN_PASSWORD を設定
```

### 2. クイズ内容の編集

`quiz-content/quiz_data.json` を編集してクイズを設定します。

```json
{
  "quiz_title": "クイズ大会",
  "theme_color": "#7C3AED",
  "welcome_image": "",
  "finished_image": "",
  "questions": [
    {
      "id": 0,
      "title": "第1問",
      "body": "問題文を入力",
      "choices": [
        { "id": "A", "image": "/images/q1_a.jpg", "label": "選択肢A" },
        { "id": "B", "image": "/images/q1_b.jpg", "label": "選択肢B" },
        { "id": "C", "image": "/images/q1_c.jpg", "label": "選択肢C" },
        { "id": "D", "image": "/images/q1_d.jpg", "label": "選択肢D" }
      ],
      "correct_answer": "A"
    }
  ]
}
```

画像ファイルは `quiz-content/images/` に配置してください。

### 3. 起動

```bash
docker compose up -d --build
```

ブラウザで http://localhost を開きます。

管理者画面: http://localhost/admin

## quiz_data.json 設定リファレンス

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `quiz_title` | ✓ | クイズのタイトル |
| `theme_color` | ✓ | テーマカラー (16進数, 例: `"#7C3AED"`) |
| `welcome_image` | — | トップページのアイコン画像パス (例: `"/images/welcome.png"`)。空欄の場合はデフォルトのSVGアイコンを表示 |
| `finished_image` | — | クイズ終了ページのアイコン画像パス。空欄の場合はデフォルトのSVGアイコンを表示 |
| `questions` | ✓ | 問題の配列 |

### 問題 (questions) の構造

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `id` | ✓ | 問題番号 (0始まりの連番) |
| `title` | ✓ | 問題タイトル (例: `"第1問"`) |
| `body` | ✓ | 問題文 |
| `choices` | ✓ | 選択肢の配列 (4つ) |
| `correct_answer` | ✓ | 正解の選択肢ID (`"A"` / `"B"` / `"C"` / `"D"`) |

### 選択肢 (choices) の構造

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `id` | ✓ | 選択肢ID (`"A"` / `"B"` / `"C"` / `"D"`) |
| `image` | — | 画像パス (例: `"/images/q1_a.jpg"`) |
| `label` | ✓ | 選択肢のラベルテキスト |

### テーマカラーの例

| カラー | コード |
|--------|--------|
| パープル (デフォルト) | `"#7C3AED"` |
| インディゴ | `"#6366F1"` |
| レッド | `"#DC2626"` |
| グリーン | `"#10B981"` |
| アンバー | `"#F59E0B"` |
| ピンク | `"#EC4899"` |

## HTTPS (TLS) 設定

1. 証明書ファイルを `nginx/certs/` に配置:
   - `fullchain.pem` — 証明書 + チェーン
   - `privkey.pem` — 秘密鍵

2. `nginx/nginx.conf` のTLSセクションをアンコメント

3. HTTPSリダイレクトブロックをアンコメント (任意)

4. コンテナを再起動:
   ```bash
   docker compose restart nginx
   ```

## 管理者操作フロー

1. `/admin` にアクセスしてパスワードでログイン
2. 参加者が `/` からアクセスして名前を登録
3. 管理者が **クイズ開始** ボタンを押す
4. 参加者に第1問が表示され、回答を受け付ける
5. 管理者が **一時停止** で回答受付を止めることができる (参加者の選択肢ボタンが無効化される)
6. 管理者が **次の問題** で次問へ進む
7. 全問終了後、参加者にスコアと問題別結果が表示される
8. 管理者画面でリアルタイムにスコアランキングを確認できる (正解数降順)

### 管理者画面の機能

| ボタン | 動作 |
|--------|------|
| クイズ開始 | 第1問から開始 |
| 一時停止 / 再開 | 回答の受付を止める / 再開する |
| 次の問題 | 次問へ進む (最終問題の場合はクイズ終了) |
| 前の問題 | 前問へ戻る |
| 問題ジャンプ | 任意の問題番号へ直接移動 |
| リセット | **全参加者データ (名前・回答) を削除**してクイズを初期状態に戻す |

### リセットについて

リセットを実行すると以下がすべて削除されます:

- Redis 上の全参加者の名前・回答データ
- 参加者のブラウザ上のローカルデータ (名前・ユーザーID)

次のクイズには再度名前の登録が必要になります。

## 参加者のクイズフロー

1. URL にアクセス → **クイズをはじめる** ボタンをタップ
2. 名前を入力して参加登録
3. 管理者がクイズを開始するまで待機画面が表示される
4. クイズ開始後、問題と4つの画像付き選択肢が表示される
5. 選択肢をタップ → **確認ダイアログ** が表示される
6. 確認後に送信 → 「回答済み」状態になり次の問題を待つ
7. 管理者が一時停止中は選択肢ボタンが無効化される
8. 全問終了後、自分のスコアと問題別結果を確認できる
